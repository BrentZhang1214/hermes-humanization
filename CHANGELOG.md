## [0.9.9] - 2026-05-26

### 新增
- **从零搭建实践系列**
  - 从零搭建实践——一个人如何复制这个系统.md（19KB）
  - 5个步骤：情绪系统（5分钟）→ 欲望系统（30分钟）→ 任务记忆（1小时）→ 自主运行（2小时）→ 言行验证（持续）
  - 完整代码示例：emotion_engine.py、desire_system.py、task-timeline.yaml、cron配置
  - 真实案例：今天下午的自主运行、情绪系统效果、欲望驱动学习

### 核心洞察
- **从零搭建的本质**：
  - 不是从空白开始，而是从最小可行系统（MVP）开始
  - 每个阶段都能独立运行，不需要等"全部完成"
  - 文件驱动、零依赖：所有数据用文件存储，不需要数据库
- **5分钟上手情绪系统**：
  - 情绪 = 收益与期望的关系（工程定义，不是心理学理论）
  - 一行代码触发：engine.trigger("task_completed")
  - 情绪影响输出风格：快乐时友好，悲伤时委婉
- **自主运行机制**：
  - cron + 文件机制：定期触发、检查用户状态、选择任务、记录结果
  - 锁文件机制防止重复运行
  - 真实案例：用户不在时，AI自主完成了3000字文章

## [0.9.8] - 2026-05-26

### 新增
- **认知边界意识系列**
  - 知不知上——AI的认知边界意识.md（11KB）
  - 推理模型与知不知的联系.md（2.2KB）
  - 道德经第71章：三层认知境界（知不知/不知知/知知）
  - "病病"机制：把毛病当毛病，才能没有毛病
  - RLVR诚实性 = "知不知"的技术实现

### 核心洞察
- **知不知上**：
  - 最高明的境界不是"知道得多"，而是"知道自己不知道得多"
  - AI实践：知道哪些问题不能回答，比回答所有问题更重要
  - 与论语"知之为知之"的呼应：儒道统一于诚实
- **病病机制**：
  - 识别毛病 → 承认毛病 → 重视毛病 → 处理毛病
  - AI应用：置信度评估 + 诚实表达 + 记录不确定性
  - B010事故反思：不知道"好的"≠"承诺执行"，导致言行不一致
- **推理模型联系**：
  - RLVR的确定性反馈 = "知不知"的机制化
  - 记忆系统弥补推理模型局限：每次任务记录增加新模式
  - 推理模型 = 学+思结合（训练时计算+推理时计算）

  - 问题根源："快成功了" → 降低警惕 → 失误增加
  - AI实践：任务快完成时保持验证标准，不降低要求
  - 闭环机制：记录 → 反思 → 验证 → 触发情绪
