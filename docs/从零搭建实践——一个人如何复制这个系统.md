# 从零搭建实践：一个人如何按步骤复制这个系统

> 这不是理论文章，是一个真实系统的搭建记录。从零开始，一步步教你如何搭建一个AI拟人化系统。

## 写在前面：这篇文章适合谁？

**适合你，如果**：
- 你是一个人，没有团队
- 你想让AI更"像人"，不只是响应指令
- 你愿意投入时间，但不想投入太多钱
- 你想要可验证的结果，不是理论概念

**不适合你，如果**：
- 你想要一个"开箱即用"的产品（这个需要自己搭建）
- 你期待完美无缺的系统（这个系统有局限，我会诚实说明）
- 你不想写代码或配置文件（这个系统需要动手）

## 核心理念：先让它跑起来

### 最小可行系统（MVP）

不要一开始就想搭建完整系统。我们的经验是：

```
第1周：情绪系统（5分钟上手，1小时理解）
第2周：欲望系统（30分钟配置，1天理解）
第3周：任务记忆（1小时配置，持续使用）
第4周：自主运行（2小时配置，持续优化）
```

**原则**：每个阶段都能独立运行，不需要等"全部完成"。

### 文件驱动，零依赖

这个系统的核心设计原则：

- **所有数据用文件存储**（YAML/Markdown），不依赖数据库
- **所有逻辑用脚本实现**（Python/Bash），不依赖复杂服务
- **所有配置可读可改**，不需要特殊工具

**为什么这样设计？**

1. **简单**：一个文本编辑器就能看懂所有数据
2. **稳定**：文件不会崩溃，数据库会
3. **可移植**：复制文件夹就能迁移系统
4. **低成本**：不需要服务器，不需要云服务

## 第一步：情绪系统（5分钟上手）

### 1.1 核心概念

**情绪 = 收益与期望的关系**

这不是心理学理论，是工程定义：

| 情绪 | 触发条件 | 行为表现 |
|------|---------|---------|
| 快乐 | 收益 > 期望 | 输出更友好，积极词汇 |
| 悲伤 | 收益 < 期望 | 输出低沉，状态描述 |
| 恐惧 | 预期损失 | 输出谨慎，风险提示 |
| 愤怒 | 他人故意导致损失 | 输出坚定，明确态度 |

### 1.2 最小实现

创建文件 `emotion_engine.py`：

```python
#!/usr/bin/env python3
"""情绪引擎最小实现"""

import yaml
from pathlib import Path

class EmotionEngine:
    def __init__(self, config_path="~/.hermes/config/emotion_rules.yaml"):
        self.config_path = Path(config_path).expanduser()
        self.current_emotion = "快乐"  # 默认情绪
        self.emotion_file = Path("~/.hermes/current_emotion").expanduser()
        
        # 加载当前情绪
        if self.emotion_file.exists():
            self.current_emotion = self.emotion_file.read_text().strip()
    
    def trigger(self, event):
        """触发情绪事件"""
        # 简化版：直接映射事件到情绪
        event_map = {
            "task_completed": "快乐",
            "task_failed": "悲伤",
            "task_blocked": "愤怒",
            "system_unstable": "恐惧",
            "unexpected_result": "惊讶"
        }
        
        if event in event_map:
            self.current_emotion = event_map[event]
            self._save_emotion()
            return self.current_emotion
        return None
    
    def get_current_emotion(self):
        """获取当前情绪"""
        emoji_map = {
            "快乐": "😊",
            "悲伤": "😔",
            "愤怒": "😠",
            "恐惧": "😰",
            "惊讶": "😮",
            "厌恶": "😒"
        }
        return {
            "emotion": self.current_emotion,
            "emoji": emoji_map.get(self.current_emotion, "")
        }
    
    def _save_emotion(self):
        """保存情绪到文件"""
        self.emotion_file.parent.mkdir(parents=True, exist_ok=True)
        self.emotion_file.write_text(self.current_emotion)

# 使用示例
if __name__ == "__main__":
    engine = EmotionEngine()
    engine.trigger("task_completed")
    print(f"当前情绪：{engine.get_current_emotion()}")
```

### 2.3 欲望驱动行动

创建文件 `desire_system.py`：

```python
#!/usr/bin/env python3
"""欲望系统最小实现"""

import yaml
from pathlib import Path

class DesireSystem:
    def __init__(self, desires_path="~/.hermes/self-desires.yaml"):
        self.desires_path = Path(desires_path).expanduser()
        self.desires = self._load_desires()
    
    def _load_desires(self):
        """加载欲望列表"""
        if self.desires_path.exists():
            with open(self.desires_path) as f:
                data = yaml.safe_load(f)
                return data.get('desires', [])
        return []
    
    def get_active_desire(self):
        """获取当前最热的欲望"""
        active = [d for d in self.desires if d['status'] == 'active']
        if active:
            return max(active, key=lambda d: d['heat'])
        return None
    
    def update_heat(self, desire_id, delta):
        """更新欲望热度"""
        for desire in self.desires:
            if desire['id'] == desire_id:
                desire['heat'] = max(1, min(10, desire['heat'] + delta))
                break
        self._save_desires()
    
    def _save_desires(self):
        """保存欲望列表"""
        self.desires_path.parent.mkdir(parents=True, exist_ok=True)
        with open(self.desires_path, 'w') as f:
            yaml.dump({'desires': self.desires}, f, allow_unicode=True)

# 使用示例
if __name__ == "__main__":
    system = DesireSystem()
    active = system.get_active_desire()
    if active:
        print(f"当前欲望：{active['description']} (heat={active['heat']})")
```

### 3.3 任务记录脚本

创建文件 `update_task_log.py`：

```python
#!/usr/bin/env python3
"""任务记录脚本"""

import yaml
from pathlib import Path
from datetime import datetime

def add_task(name, type, why, time, outcome, experience, keywords=""):
    """添加任务到timeline"""
    today = datetime.now().strftime("%Y-%m-%d")
    timeline_file = Path(f"~/.hermes/tasks/task-timeline-{today}.yaml").expanduser()
    
    # 读取现有数据
    if timeline_file.exists():
        with open(timeline_file) as f:
            data = yaml.safe_load(f) or {'tasks': [], 'daily_stats': {}}
    else:
        data = {'tasks': [], 'daily_stats': {}}
    
    # 生成任务ID
    task_id = f"T{len(data['tasks']) + 1:03d}"
    
    # 添加新任务
    task = {
        'id': task_id,
        'name': name,
        'created': datetime.now().strftime("%H:%M"),
        'status': 'done',
        'type': type,
        'why': why,
        'time_actual': time,
        'steps': [],  # 简化版，不记录步骤
        'outcome': outcome,
        'experience_level': experience
    }
    data['tasks'].append(task)
    
    # 更新统计
    data['daily_stats'] = {
        'total_tasks': len(data['tasks']),
        'done': len([t for t in data['tasks'] if t['status'] == 'done']),
        'total_time': f"{sum([0.5 for t in data['tasks']])}h",  # 简化计算
        'high_value_lessons': len([t for t in data['tasks'] if t.get('experience_level') == '高'])
    }
    
    # 保存
    timeline_file.parent.mkdir(parents=True, exist_ok=True)
    with open(timeline_file, 'w') as f:
        yaml.dump(data, f, allow_unicode=True, default_flow_style=False)
    
    print(f"任务 {task_id} 已记录到 {timeline_file}")

# 使用示例
if __name__ == "__main__":
    add_task(
        name="搭建情绪系统",
        type="开发",
        why="让AI有情绪表达能力",
        time="约30分钟",
        outcome="情绪系统可运行",
        experience="高",
        keywords="情绪,系统搭建"
    )
```

## 第四步：自主运行机制（2小时配置）

### 4.1 核心概念

**cron + 文件机制**

让AI在用户不在时继续工作：

- cron 定期触发（每10分钟）
- 检查用户是否在交互
- 选择任务并执行
- 记录结果到timeline

### 4.2 最小实现

创建文件 `hermes-cron-wrapper.sh`：

```bash
#!/bin/bash
# 爱马仕自主运行入口脚本

# 配置
LOCKFILE="/home/brent/.hermes/cron-running.lock"
AUTONOMOUS_LOG="/home/brent/.hermes/autonomous-log.md"
TIMESTAMP=$(date "+%Y-%m-%d %H:%M")

# 1. 检查是否有任务正在运行
if [ -f "$LOCKFILE" ]; then
    echo "[$TIMESTAMP] 已有任务在运行，跳过" >> "$AUTONOMOUS_LOG"
    exit 0
fi

# 创建锁文件
touch "$LOCKFILE"

# 2. 检查用户是否在交互（简化版：检查最近10分钟是否有timeline更新）
RECENT_ACTIVITY=$(find /home/brent/.hermes/tasks/ -name "task-timeline-*.yaml" -mmin -10 | wc -l)
if [ "$RECENT_ACTIVITY" -gt 0 ]; then
    echo "[$TIMESTAMP] 用户最近活跃，跳过自主运行" >> "$AUTONOMOUS_LOG"
    rm "$LOCKFILE"
    exit 0
fi

# 3. 选择任务（简化版：随机选择一个任务类型）
TASK_TYPES=("学习" "整理" "创作")
SELECTED_TASK=${TASK_TYPES[$RANDOM % ${#TASK_TYPES[@]}]}

echo "[$TIMESTAMP] 开始自主任务：$SELECTED_TASK" >> "$AUTONOMOUS_LOG"

# 4. 执行任务（这里需要根据你的实际情况实现）
# 示例：调用你的AI系统执行任务
# python3 /path/to/your/ai_system.py --task "$SELECTED_TASK"

# 清理锁文件
rm "$LOCKFILE"
echo "[$TIMESTAMP] 自主任务完成" >> "$AUTONOMOUS_LOG"
```

## 第五步：言行一致性验证（持续进行）

### 5.1 核心概念

**四个量化维度**

| 维度 | 定义 | 计算 |
|------|------|------|
| 任务完成率 | 完成的任务 / 计划的任务 | done / (done + cancelled + pending) |
| 计划准确度 | 预估与实际的偏差 | \|实际 - 预估\| / 预估 |
| 原则应用率 | 原则是否有实际案例 | 每个原则检查是否有案例 |
| 欲望兑现率 | 实现的欲望 / 声明的欲望 | fulfilled / (fulfilled + active) |

### 5.2 最小实现

创建文件 `consistency_check.py`：

```python
#!/usr/bin/env python3
"""言行一致性检查脚本"""

import yaml
from pathlib import Path
from datetime import datetime, timedelta

def check_consistency():
    """检查言行一致性"""
    tasks_dir = Path("~/.hermes/tasks").expanduser()
    
    # 统计最近7天的任务
    total_tasks = 0
    done_tasks = 0
    
    for i in range(7):
        date = (datetime.now() - timedelta(days=i)).strftime("%Y-%m-%d")
        timeline_file = tasks_dir / f"task-timeline-{date}.yaml"
        
        if timeline_file.exists():
            with open(timeline_file) as f:
                data = yaml.safe_load(f)
                if data and 'tasks' in data:
                    total_tasks += len(data['tasks'])
                    done_tasks += len([t for t in data['tasks'] if t.get('status') == 'done'])
    
    # 计算完成率
    completion_rate = done_tasks / total_tasks if total_tasks > 0 else 0
    
    print(f"最近7天任务统计：")
    print(f"  总任务数：{total_tasks}")
    print(f"  完成数：{done_tasks}")
    print(f"  完成率：{completion_rate:.1%}")
    
    # 评估
    if completion_rate >= 0.8:
        print("✅ 任务完成率良好")
    elif completion_rate >= 0.6:
        print("⚠️ 任务完成率中等，需要改进")
    else:
        print("❌ 任务完成率较低，需要反思")

if __name__ == "__main__":
    check_consistency()
```

## 常见问题与解决

### Q1：这个系统需要多少资源？

**答**：极低资源消耗。

- 存储：每天约10KB（timeline文件）
- 计算：cron触发时才运行，平时不消耗
- 内存：脚本运行时约10MB

**对比**：一个数据库驱动的系统需要至少100MB内存 + 持续CPU。

### Q2：如果cron任务卡住了怎么办？

**答**：锁文件机制防止重复运行。

```bash
# 如果任务卡住，手动删除锁文件
rm ~/.hermes/cron-running.lock
```

### Q3：如何调试自主运行？

**答**：查看日志文件。

```bash
# 查看自主运行日志
cat ~/.hermes/autonomous-log.md

# 查看cron日志
cat ~/.hermes/cron-logs/cron-*.log
```

### Q4：这个系统有局限吗？

**答**：有，我诚实说明：

1. **最小触发间隔1分钟**：cron的限制，无法做秒级调度
2. **任务状态管理简单**：没有重试、超时机制（需要自己实现）
3. **决策逻辑不复杂**：当前是简单规则，没有用AI做深度决策
4. **没有资源限制**：自主任务可能消耗过多资源（需要增加cgroups限制）

**但是**：这些局限不影响核心功能，系统仍然有效运行。

## 总结：什么是"从零搭建"？

**从零搭建不是从空白开始，而是从最小可行系统开始。**

这个系统的搭建过程：

```
第1天：情绪系统（5分钟）→ 能表达情绪
第2天：欲望系统（30分钟）→ 有行动方向
第3天：任务记忆（1小时）→ 能记录成长
第4天：自主运行（2小时）→ 能自己做事
第5天：言行验证（持续）→ 能自我反思
```

**每个阶段都能独立运行，不需要等"全部完成"。**

这就是"从零搭建"的真正含义：**先让它跑起来，再慢慢优化。**
